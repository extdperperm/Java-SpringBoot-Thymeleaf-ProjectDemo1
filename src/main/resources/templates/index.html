Proyecto

#Spring Boot Project Demo 1

Objetivos

- Comprender la estructura y elementos de un proyecto web con el gestor de proyecto MAVEN.

- Configuración básica de un proyecto MVC Web con Spring Boot.

- Familiarizarse con la arquitectura Modelo Vista Controlador de Spring.

- Ser capaz de crear las controladoras (@Controller).

- Dominar el mapeo de direcciones o urls (@GetMapping, @PostMapping).

- Plantear diversas formas de envío y recogida de parámetros con GET y POST (@RequestParam).

- Comprender y hacer uso del Modelo en Spring (Model)

- Entender y saber usar las notaciones básicas en Expression language (EL) para renderizar los datos en las vistas con Thymeleaf

Entorno de Desarrollo

Ide: Sprint Tool Suite, versión: 4

Servidor de referencia: Apache Tomcat, versión: 10

Jdk: OpenJdk, versión: 17.1

Estandar de Desarrollo

Gestor de proyecto: Maven, versión: 4

Dependencias

Spring Boot Framework: Spring, versión 3.1.*. Módulos: spring-boot-starter-web, spring-boot-starter-tomcat, tomcat-embed-jasper, tomcat-embed-jasper

Es decir, las vistas se pueden organizar físicamentes en difernetes niveles de carpetas. Haz clic aquí

Ejemplo 6 - Envios de datos POST: Formulario que envía por método POST a la ruta relativa ./ejemplo6 (controladora MainController) un número que se recoge con anotaciones @RequestParam (Ver código de la controladora), se trata sumandole 5 y se carga en el Modelo de Spring un nuevo atributo para que alimente la vista que mostrará el resultado. Tenga en cuenta que da igual que el método de envío sea GET o POST, pues la forma de recoger los parámetros puede ser la misma.

Ejemplo 7 - Envios de datos POST: Formulario que envía por método POST varios datos del alumno a la ruta relativa ./ejemplo7 (controladora MainController) y se recoge con anotaciones @RequestParam (Ver código de la controladora), se crea un objeto Alumno (modelo de negocio) que a su vez se carga en el Modelo de Spring que alimentará la vista.

Ejemplo 8 - Envios de datos POST: Formulario que envía por método POST varios datos del alumno a la ruta relativa ./ejemplo8 (controladora MainController) y se recoge con un solo parámetro de tipo MAP con la anotación @RequestParam (Ver código de la controladora), se crea un objeto Alumno (modelo de negocio) que a su vez se carga en el Modelo de Spring que alimentará la vista. Este ejemplo muestra como se pueden recoger un número indeterminados de parámetros (múltiples parámetros) sin especificar por cada parámetro una @RequestParam.

En este caso, además, tiene una muestra de como Thymeleaf permite utilizar estructuras de control con las que se puede decidir en base a una condición si se muestra o no código html.
